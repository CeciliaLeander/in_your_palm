# 文档五 · UI 改造规范

## 5.1 UI 改造全景

### 5.1.1 需要改动的界面

| 界面 | 改动程度 | 原因 |
|---|---|---|
| **开场菜单** | 新增第 9 步(人格配置) | 人格模板系统 |
| **右侧抽屉** | 结构重组 + 新增珠谱区 + 刻印警示条 | 适配新架构 |
| **动作按钮面板** | 基本保留,但**移除原"调教"大类**,重新划分 | 新的 9 大动作分类 |
| **数值区** | **大改**:移除 arousal/shame/trained,新增会话面板 | 数值架构变了 |
| **会话结算弹窗** | **全新** | 三层架构的关键仪式 |
| **珠谱折叠区** | **全新** | 珠的展示 |
| **刻印警示条** | **全新** | 刻印的存在感 |

### 5.1.2 设计原则

1. **仪式感 vs 功能感分离**
   - 功能性信息(生理值、当前动作) → 紧凑、常驻
   - 仪式性信息(珠解锁、刻印烙印、会话结算) → 有动效、有视觉冲击

2. **默认折叠,按需展开**
   - 珠谱默认折叠(避免喧宾夺主)
   - 刻印警示条常驻但薄(一行即可)
   - 会话面板**仅在会话中**显示

3. **三套主题保持一致性**
   - noir(暗黑监控)
   - ornate(暗金奢华)
   - raw(粗粝铁锈)
   - 所有新元素都要在三套主题下都有设计

## 5.4 数值区 · 详细设计

### 5.4.1 显示逻辑

**三类分组**(原 11 项 → 新 8 项):

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
心理  理智 62  心情 40  真心 35  友好 28
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
生理  体力 55  饥饿 30  睡眠 45  健康 85
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
状态  🕯清醒  🔗束缚中  📿戴项圈
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**被移除的**:
- ❌ arousal(被 pleasure/desire palam + 欲望珠取代)
- ❌ shame(被 shame palam + 羞耻珠取代)
- ❌ trained(被 submission palam + 服从珠取代)
- ❌ distortion(被 distortion_palam + 扭曲珠取代,且隐藏)

这 4 项现在**不在数值区显示**——它们是"瞬时/累积"值,不是"基础状态"。
- **瞬时**值(palam)在会话面板显示
- **累积**值(珠)在珠谱显示

### 5.4.2 视觉设计

**数值的等级颜色**(3 主题通用):
- 0-20:红色(临界)
- 21-40:橙色(低)
- 41-70:中性色(正常)
- 71-85:浅绿(高)
- 86-100:绿色(饱满)

**数值变化动画**:
- 增加:数字上方短暂出现 +3 等绿色浮字,0.8s 消失
- 减少:数字上方短暂出现 -5 等红色浮字
- 剧烈变化(单次变化 > 10):数值本身闪烁一下

**状态徽章**(第 3 行):
- 图标化呈现,不显示数值
- 点击徽章弹出 tooltip 说明

## 5.5 会话面板 · 全新元素

### 5.5.1 显示条件

- `STATE.session.active === true` 时显示
- 非会话状态时**完全隐藏**(不留空位)

### 5.5.2 视觉设计

```
┌─────────────────────────────────────────┐
│  ⚫ 会话中 (pulsing indicator)            │
│  持续 32 分钟 · 已执行 4 个动作           │
│  ─────────────────────────────────────  │
│  本次累积:                                │
│    服从 45 ████████░░ (触及铜级)         │
│    羞耻 32 ██████░░░░                    │
│    扭曲 ●● (隐藏数值)                    │
│    欲望 12 ██░░░░░░░░                    │
│    空虚 8  █░░░░░░░░░                    │
│    反感 23 ████░░░░░░                    │
│  ─────────────────────────────────────  │
│  最近动作: 训练坐姿                       │
│  产生: 屈従+3, 羞耻+2, 疲劳+4             │
│                                         │
│           [ 结束本次调教 ]                │
└─────────────────────────────────────────┘
```

### 5.5.3 关键视觉细节

**呼吸动画**
头部的"⚫ 会话中"圆点用 1.5s 的呼吸节律闪烁,让玩家潜意识感知到"正在进行中"。

**Palam 进度条**
- 每个 palam 有一条进度条,满格 = 触发一颗珠(100 palam)
- 超过 100 时,条变色并显示"触及铜级"之类的提示

**扭曲 palam 的隐藏**
- 数值位置显示 `●●`(小圆点,视觉上示意"有内容但看不到")
- 进度条不显示
- 鼠标悬停显示 tooltip:"你似乎感觉到了什么变化,但无法名状"
- **这是本设计的精华**:玩家知道有事在发生,但不知道走到哪了

**"结束本次调教"按钮**
- 显眼但不激进
- 点击后**不直接结束**,先弹确认:"确定结束?本次累积将转化为珠。"
- 确认后触发结算流程(见 §5.6)

### 5.5.4 "最近动作"区

显示上一次动作产生的 source(**原始值**,未经转化):
```
最近动作: 训练坐姿
产生: 屈従+3, 羞耻+2, 疲劳+4, 不快+1
```

**为什么显示 source 而不是 palam 增量?**
- source 是"**客观事实**"(动作产生了什么)
- palam 是"**主观反应**"(char 因为人格/状态如何消化这些)
- source 让玩家看到"**我做了什么**",palam 看到"**char 如何反应**"
- 两个视角都有,玩家才能学会"**读**" char 的反应

## 5.6 会话结算弹窗 · 核心仪式

### 5.6.1 触发时机

会话结束时,弹出结算弹窗。**必须响应,不能跳过**(这是核心叙事时刻)。

### 5.6.2 弹窗结构

```
╔═══════════════════════════════════════════╗
║                                           ║
║            ━━ 本次调教 ━━                  ║
║                                           ║
║  持续 2 小时 15 分钟 · 执行 7 个动作        ║
║                                           ║
║  ─────────────────────────────────────    ║
║                                           ║
║  本次累积转化为:                           ║
║                                           ║
║    ● 服从珠 +1                             ║
║    ● 羞耻珠 +1                             ║
║    ● 反抗珠 +2                             ║
║                                           ║
║  ─────────────────────────────────────    ║
║                                           ║
║  🏆 解锁新等级!                            ║
║                                           ║
║    ● 服从珠 · 铜级「初现屈服」              ║
║                                           ║
║    ❝ char 在强压下会出现短暂的顺从表现, ❞  ║
║    ❝ 但本质上仍保有抗拒意志。          ❞   ║
║                                           ║
║  ─────────────────────────────────────    ║
║                                           ║
║             [  继续  ]                     ║
║                                           ║
╚═══════════════════════════════════════════╝
```

### 5.6.3 特殊场景

**场景 1:扭曲珠悄悄出现**
- 结算时**不显示**扭曲珠的获得
- 但后台悄悄加上
- 玩家下次翻看珠谱时会发现"咦,什么时候多了一颗扭曲珠?"
- **这是本设计的精髓**,不能妥协

**场景 2:等级晋升**
- 有新等级解锁时,中间加一段"🏆 解锁新等级"
- 黑曜级解锁时,加强版本:弹窗背景闪烁,文字用金色/更深色(按主题)
- 晋升文本完整展示印痕(Part 2 §3.5-3.10 的文本)

**场景 3:多重等级跳跃**
- 如果本次累积直接跨过多个等级(比如从无 → 铜级 → 银级),**依次展示**
- 每级解锁的提示各 1.5s 渐入,玩家可以 tap 跳过

**场景 4:触发了刻印**
- 结算中途穿插"**一道刻痕被烙下了**"的提示(见 §5.7.3)
- **优先于等级解锁**展示(刻印是"事故",等级是"成就",事故先呈现)

### 5.6.4 视觉设计差异化(三主题)

**noir 主题**:
- 背景:深黑 + 细线边框
- 文字:冷白色
- 高亮:银灰色
- 动效:细腻、克制、像高冷监控画面

**ornate 主题**:
- 背景:深金 + 金色雕花边框
- 文字:米白色
- 高亮:暗金色
- 动效:如打开一本古旧账本

**raw 主题**:
- 背景:铁锈棕 + 粗糙笔触边框
- 文字:旧羊皮纸色
- 高亮:血红色
- 动效:像写字机敲击文字逐字出现

## 5.9 动作按钮面板 · 9 个 tab 分组

### 5.9.1 Tab 结构

由于有 13 个类别,分两行 tab:

**第一行(常用)**:
```
[环境] [递送] [接触] [触碰] [训练] [束缚] [刺激]
```

**第二行(次要)**:
```
[羞辱] [护理] [医疗] [状态] [外出] [自定义]
```

### 5.9.2 Tab 标签样式

每个 tab 标签附带**会话触发指示器**:
- ● 红点(启动会话类):触碰、训练、束缚、刺激、羞辱
- ○ 空心圆(不启动):环境、递送、接触、护理、医疗、状态、外出、自定义

```
[环境 ○] [递送 ○] [接触 ○] [触碰 ●] [训练 ●] [束缚 ●] [刺激 ●]
                                    [羞辱 ●] [护理 ○] [医疗 ○] [状态 ○] [外出 ○] [自定义 ○]
```

**点击动作时**:
- 如果是启动会话类且没在会话中 → 开始会话
- 如果在会话中且点击非会话类 → 不影响会话(除非是 state_change 类的 put_to_sleep)

### 5.9.3 按钮面板内的动作组织

每个 tab 下,动作按**子分类**组织(不是平铺):

## 5.11 响应式 / 尺寸考虑

### 5.11.1 抽屉宽度

**桌面**:480px(比当前 440px 略宽,因为多了珠谱和会话面板)

**移动端**:建议**全屏抽屉**,侧滑进入全屏(这是 v0.6.0 之后的优化方向,本期不做)

### 5.11.2 内容溢出处理

当有 3+ 刻印、6 颗珠解锁、会话面板活跃时,抽屉会**很长**。

- 整个抽屉内部**可滚动**
- 头部(标题+信息条+刻印警示)**粘性置顶**,滚动时仍在视野
- 底栏(重新配置/结束本局)**粘性置底**
